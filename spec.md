# 快递查询微信小程序 — 产品规格文档 (spec.md)

> V1 版本 | 2026-04-25

---

## 1. 项目概述

### 1.1 产品定位
一款快递物流查询微信小程序，用户输入快递单号即可查询实时物流轨迹。

### 1.2 核心价值
- **用户侧**：一键查快递，无需下载 App，无需注册
- **商业侧**：通过微信小程序广告（Banner + 激励视频）变现
- **增长引擎**：SEO/GEO 搜索流量，覆盖快递查询长尾关键词（注：搜索收录和排序由微信平台决定，非确定性结果，V1 仅做基础配置，不作为硬性交付指标）

### 1.3 技术栈
| 层 | 方案 |
|---|---|
| 前端 | 微信小程序（Skyline 渲染引擎） |
| 后端 | 微信云开发（云函数 + 云数据库） |
| 数据源 | 快递100实时查询 API |
| 广告 | 微信小程序广告组件 |

## 2. UI 设计规范

### 2.1 风格
微信原生风，使用 WeUI 组件规范，零学习成本。

### 2.2 配色（绿色自然系）
| 用途 | 色值 | CSS 变量 | 说明 |
|---|---|---|---|
| 主色 | `#111815` | `--color-primary` | 正文标题、导航栏文字 |
| 品牌色 | `#1F6B45` | `--color-brand` | 查询按钮、状态高亮、品牌元素 |
| 页面背景 | `#F5F7F5` | `--color-bg` | 浅灰底色（绿色调） |
| 卡片背景 | `#FFFFFF` | `--color-card` | 内容卡片 |
| 辅助文字 | `#5F6B63` | `--color-text-secondary` | 次要信息 |
| 边框 | `#E3E8E3` | `--color-border` | 分割线、输入框边框 |

所有颜色通过 `app.wxss` 中 `page` 选择器的 CSS custom properties 定义，页面样式使用 `var(--color-*)` 引用。

### 2.3 排版
- 字体：微信原生字体栈（不引入外部字体）
- 正文：28rpx（14px）
- 标题：32rpx（16px），bold
- 大标题：36rpx（18px），bold
- 间距：页面左右 padding 32rpx，通过 `--space-*` 变量体系管理
- 圆角：`--radius-sm: 12rpx`, `--radius-md: 20rpx`, `--radius-lg: 28rpx`

### 2.4 组件规范
- 按钮：圆角 12rpx（`--radius-sm`），查询按钮使用品牌色背景 + 白色文字
- 输入框：白底，1rpx `--color-border` 边框，圆角 12rpx，高度 88rpx
- 卡片：白底，圆角 20rpx（`--radius-md`），无阴影
- 列表项：高度 100rpx，底部分割线
- 不使用自定义图标库，使用微信原生 icon 或纯文字

## 3. 页面结构

### 3.1 首页（查询页）`pages/index/index`

**页面元素（从上到下）：**

1. **导航栏**：标题"快递查询"
2. **剪贴板提示**：检测到剪贴板有疑似单号时，顶部显示提示条，点击可自动填充
3. **输入区域**（白色卡片）：
   - 快递单号输入框（placeholder: "请输入快递单号"）
   - 自动识别公司标签（基于 `constants.js` 的 `detectCompany()` 前缀匹配）
   - 手机号后4位输入框（仅当查询返回 `PHONE_REQUIRED` 错误时显示）
   - 查询按钮（绿色品牌色背景"查询"）
4. **查询历史区域**：
   - 标题"查询历史" + 清空按钮
   - 历史列表：每条显示单号 + 快递公司名 + 最新状态标签 + 查询时间
   - 点击可快速重新查询，左滑或长按可删除单条
   - 相对时间格式："今天 HH:MM"、"昨天 HH:MM"、"MM-DD HH:MM"
   - 无历史时显示空状态提示

**页面标题**：快递查询 - 物流查询助手

**交互逻辑：**
- 输入单号后点击查询 → 跳转结果页（携带 nu, com, phone 参数）→ 结果页调用云函数
- 按钮防抖：点击后 loading，期间不可重复点击
- 单号格式校验：6-30位字母数字和连字符（`/^[A-Za-z0-9\-]{6,30}$/`）
- 公司识别：前端通过 `detectCompany()` 本地前缀匹配，未匹配则传 `com=auto` 由后端自动识别
- 手机号后4位：默认不显示；当结果页查询返回 `PHONE_REQUIRED` 时，结果页内联显示手机号输入框
- 剪贴板检测：页面 `onShow` 和输入框 `onFocus` 时读取剪贴板，匹配单号格式则显示提示条
- 次数用完时：弹窗提示"今日查询次数已用完"（V1 无广告，直接提示）

## 4. 后端设计

### 4.1 云函数

#### `query` — 快递查询

**请求参数：**
```json
{
  "num": "SF1234567890",
  "com": "auto",
  "phone": "1234"  // 可选，顺丰必填
}
```

**处理流程：**
```
1. 获取调用者 openid（云函数自动注入）
2. 参数校验（单号格式、必填项）
3. 查询云数据库 rate_limit，检查今日查询次数
   - count >= 10 + bonus → 返回 { code: "LIMIT_EXCEEDED" }
4. 检查单号级缓存 query_cache
   - 缓存存在且未过期（30分钟内）→ 直接返回缓存结果，扣 count + 1
   - 缓存不存在或已过期 → 继续步骤 5
5. 构造快递100请求：
   - param = JSON.stringify({ com, num, phone })
   - sign = MD5(param + key + customer).toUpperCase()（注意：快递100要求32位大写十六进制）
   - POST poll.kuaidi100.com/poll/query.do
6. 更新 rate_limit：count + 1, lastQueryTime = now
7. 写入 query_cache（以单号为 key，TTL 30分钟）
8. 返回查询结果
```

**返回格式（成功）：**
```json
{
  "code": "OK",
  "data": {
    "nu": "SF1234567890",
    "com": "shunfeng",
    "state": "3",
    "status": "200",
    "traces": [
      { "time": "2026-04-25 14:30:00", "context": "快件已签收" }
    ]
  }
}
```

**返回格式（错误码完整列表）：**
```json
{ "code": "LIMIT_EXCEEDED", "message": "今日查询次数已用完" }
{ "code": "INVALID_PARAM", "message": "快递单号格式不正确" }
{ "code": "PHONE_REQUIRED", "message": "该快递公司需要手机号后4位验证" }
{ "code": "QUERY_ERROR", "message": "快递100返回的具体错误信息" }
```

#### `reward` — 广告奖励

**防刷机制：**
- 每个用户每日最多领取3次奖励（即每日上限 10 + 15 = 25次）
- 前端生成随机 nonce（UUID），云函数校验 nonce 是否已使用（防重放）
- V1 采用客户端验证 + 服务端防重放；V2 接入微信广告服务端回调做完整验证

**请求参数：**
```json
{
  "nonce": "uuid-generated-by-client"
}
```

**处理流程：**
```
1. 获取 openid
2. 校验 nonce：查询 reward_nonces 集合，若已存在则拒绝（防重放）
3. 查询 rate_limit：
   - 若 rewardCount >= 3 → 返回 { code: "REWARD_LIMIT" }
4. bonus += 5, rewardCount += 1
5. 记录 nonce 到 reward_nonces 集合
6. 更新 rate_limit
7. 返回新的可用次数
```

**返回格式：**
```json
{ "code": "OK", "remaining": 5 }
{ "code": "REWARD_LIMIT", "message": "今日广告奖励次数已用完" }
{ "code": "DUPLICATE_NONCE", "message": "无效请求" }
```

### 4.2 云数据库

#### `rate_limit` 集合

| 字段 | 类型 | 说明 |
|---|---|---|
| `_id` | string | 自动生成 |
| `_openid` | string | 用户 openid（云函数自动写入） |
| `date` | string | 当天日期 "YYYY-MM-DD" |
| `count` | number | 今日已查询次数 |
| `bonus` | number | 广告奖励额外次数 |
| `rewardCount` | number | 今日已领取广告奖励次数（上限3） |
| `lastQueryTime` | number | 上次查询时间戳（ms） |

**规则：**
- 每日可用次数 = 10 + bonus（bonus 上限 15，即 rewardCount 上限 3）
- 每日首次查询时创建记录（date = 当天）
- 查询时先检查 date 是否为当天，非当天则重置 count=0, bonus=0, rewardCount=0, date=当天
- 无需定时清理，历史记录不影响逻辑

**数据权限：** 仅管理员可读写（云函数有权操作，前端不可直接操作）

#### `query_cache` 集合（单号级缓存）

| 字段 | 类型 | 说明 |
|---|---|---|
| `_id` | string | 快递单号（作为唯一 key） |
| `result` | object | 快递100返回的完整结果 |
| `com` | string | 快递公司代码 |
| `state` | string | 物流状态 |
| `createdAt` | number | 缓存写入时间戳（ms） |

**规则：**
- 同一单号30分钟内的重复查询直接返回缓存，不调用快递100 API
- 避免触发快递100"同一单号查询频率至少半小时"的限制
- 不同用户查同一单号也走缓存（节省 API 调用）
- 缓存过期后下次查询重新调用 API 并更新缓存

**数据权限：** 仅管理员可读写

#### `reward_nonces` 集合（防重放）

| 字段 | 类型 | 说明 |
|---|---|---|
| `_id` | string | nonce（UUID，客户端生成） |
| `_openid` | string | 使用该 nonce 的用户 |
| `createdAt` | number | 使用时间戳 |

**规则：**
- 每个 nonce 只能使用一次
- 每日重置（按 createdAt 清理前一天记录，或忽略旧记录）
- 前端每次点"看广告"时生成新 UUID 作为 nonce

## 6. 广告方案

> **V1 推迟**：流量主资质暂未满足（需累计独立访客 >= 1000），广告功能推迟到 V2。V1 仅做纯查询工具，次数用完后直接提示"明天再来"。

### 6.1 广告位配置（V2 实现）

| 位置 | 类型 | 组件 | 展示条件 |
|---|---|---|---|
| 首页底部 | Banner | `<ad unit-id="xxx">` | 始终展示 |
| 结果页底部 | Banner | `<ad unit-id="xxx">` | 始终展示 |
| 次数用完时 | 激励视频 | `<rewarded-video-ad>` | 用户主动触发 |

### 6.2 激励视频流程（V2 实现）
```
用户点击查询 → 提示次数用完 → 弹窗：
  "今日查询次数已用完，观看广告可获得额外5次查询"
  [看广告获取] [以后再说]
                                    ↓ 点击看广告
                          调用 rewardedVideoAd.show()
                                    ↓ 观看完成回调
                          调用 reward 云函数
                                    ↓
                          提示"已获得5次额外查询"
                          自动执行刚才的查询
```

### 6.3 流量主资质
- 微信小程序开通流量主需要：累计独立访客 ≥ 1000
- **初期策略**：广告位代码预留，通过 `isFlowOwner` 变量控制是否展示
- 达到1000访客后开启广告展示

## 7. 风控策略

### 7.1 频次限制
| 维度 | 限制 | 实现位置 |
|---|---|---|
| openid/天 | 10次 + 广告奖励（每日最多领3次，上限25次） | 云函数 → 云数据库 rate_limit |
| 单号级缓存 | 同一单号30分钟内走缓存，不调快递100 API | 云函数 → 云数据库 query_cache |
| 广告奖励防刷 | nonce 防重放 + 每日最多领3次 | 云函数 → 云数据库 reward_nonces + rate_limit |
| 前端防抖 | 3秒 | 前端按钮 loading |

### 7.2 参数校验
- 快递单号：6-30位，允许字母、数字、连字符（-）
- 手机号后4位：4位纯数字（顺丰时必填）

### 7.3 安全措施
- 快递100 API Key 仅存在于云函数环境变量中，前端不可见
- 云数据库设置权限：仅管理员可读写
- 所有查询通过云函数中转，前端不直接调用快递100
